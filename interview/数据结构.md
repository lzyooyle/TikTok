## 关于链表，你该了解这些！
+ 链表是通过指针串联的线性结构，分为数据域和指针域，指针域存放指向下一个节点的指针，最后一个节点指向null，第一个节点是头节点
+ 数组和链表的差别：数组的查询时间复杂度是O(1)，增删时间复杂度是O(n)，适合数据量固定，较多查询，较少增删的场景
+ 链表的查询复杂度是O(n)，增删的时间复杂度是O(1)，适合数据量不固定，较多增删，较少查询的场景
## 红黑树是满足如下条件的二叉查找树
+ 根节点式黑色，红色节点不能连续（子节点和父节点都是黑色），任何一个节点到null的黑色节点个数相同

## 3.1 B树和B+树的区别
+ 1.B树的每一个节点都存储键和数据，B+树只有在叶子节点存储数据，B+树的非叶子节点只存储键而不是数据，所以一个节点可以存储
+ 更多的键，你可以使B+树比B树更短（IO的次数等于树的高度），所以跟磁盘交换的io次数越少
+ 2.所有的B+树的叶子节点，都通过一个有序链表相连，所有的记录可以通过主键遍历排序，范围搜索的效果更好，因为数据是有序
+ 相连的，更容易搜索，B树的每一层都需要递归遍历，相邻的元素在内存中可能不相邻，缓存命中率不如B+树好
+ 3.B+树所有的查询需要从根节点到叶子节点搜索，所以搜索效率更稳定，另一方面，在B树中，每个节点都能找到数据，需要一直在叶子节点和
+ 非叶子节点之间往返移动，搜索效率低

## 3.2 红黑树与平衡二叉树的区别
+ 1.平衡二叉树靠左子树和右子树高度差绝对值不超过1来维持的，红黑树基于搜索路径上黑色节点和红色节点的数量关系来维持平衡的二叉树
+ 2.为了确保左子树和右子树的高度差，平衡二叉树在插入或删除的时候，会进行旋转，旋转次数的复杂度根据数据的不同
+ 因此在插入或删除节点是，旋转的次数是未知的，在较少的插入或修改的情况下，会引起效率问题，红黑树的旋转和改变颜色（红色转黑色，
+ 黑色转红色）在插入的时候，确保没有路径是其它路径的两倍长，总体上说，当插入或删除节点的时候，红黑树旋转的次数要小于
+ 平衡二叉树，所以红黑树适合于较多插入和删除的操作
